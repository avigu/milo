# Message Markers

Use these markers inside Slack thread messages when needed.

## Standard role prefixes
- `[PM]`
- `[Architect]`
- `[Developer]`
- `[Investment]`
- `[Milo]`

## Structured inline markers
- `STAGE: ...`
- `QUESTION_FOR_AVI: ...`
- `DECISION: ...`
- `BLOCKER: ...`
- `NEXT_STEP: ...`
- `DRIVE: <link>`

## Examples
`[PM] QUESTION_FOR_AVI: Should v1 support index presets in addition to explicit universe tickers?`

`[Architect] BLOCKER: Request shape is still ambiguous, so cache keys and orchestration boundaries should not be finalized yet.`

`[PM] DECISION: includeFailedChecks will return a separate failedCandidates section.`

`[Developer] NEXT_STEP: I will draft the execution plan after architecture is locked.`
