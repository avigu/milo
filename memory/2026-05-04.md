## myfinapp — bulk-prices / local-cache feature work

Branch: `feature/bulk-prices-local-cache` (uncommitted).

### What landed
- `utils/cache.js` — added local-file fallback. When `GCS_BUCKET` is empty/inaccessible, persistent cache writes to `<repo>/.data/cache/` (override via `LOCAL_CACHE_DIR`). Public API unchanged.
- `services/marketCap.js` — replaced thin wrapper with a real cache layer: per-ticker TTL = 7 days, persisted via `cache.js`. Sources: Finnhub `/stock/profile2` primary, FMP `/stable/quote` fallback. Exposed `recordMarketCap[s]()` so quote flows can populate it for free. Cold-start pacing: 5 concurrent + 1.1 s inter-chunk → stays under Finnhub free tier (60/min).
- `services/stockDataProvider.js` — `getBatchQuotes` now Finnhub-primary in parallel chunks of 10 (200 ms inter-chunk). FMP becomes fallback for stragglers. Observed market caps from FMP path are pushed into `marketCap` cache automatically.
- `services/opportunities.js` — both `getInvestmentOpportunities` and `getUpcomingRelevantEarnings` pre-filter by market cap before the quote round-trip; the working set typically drops from ~195 → ~30–50. Upcoming earnings flow no longer fetches quotes at all (only market caps, since it doesn't need price).
